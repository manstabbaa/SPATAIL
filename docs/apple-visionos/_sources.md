# Apple visionOS / Spatial Computing — Documentation Sources

A source index for the SPATAIL session. Apple calls XR "spatial computing," so
its guidance lives under visionOS, not under "XR." This corpus is captured to
`docs/apple-visionos/` for offline use while building the spatial education app.

> FETCHING NOTE: developer.apple.com pages are JS-rendered. For `/documentation/`
> and `/tutorials/` paths, fetch the JSON endpoint by inserting the page path
> after `https://developer.apple.com/tutorials/data`, e.g.
> <https://developer.apple.com/tutorials/data/documentation/visionos.json>

## 1. Human Interface Guidelines — design best practices (start here)
- Designing for visionOS: https://developer.apple.com/design/human-interface-guidelines/designing-for-visionos
- Spatial layout: https://developer.apple.com/design/human-interface-guidelines/spatial-layout
- HIG home: https://developer.apple.com/design/human-interface-guidelines

Most relevant HIG topics: spatial layout, immersive experiences, materials,
eyes (gaze input), gestures, motion/comfort.

## 2. Developer documentation — technical build guidance
- visionOS hub: https://developer.apple.com/visionos/
- visionOS docs root: https://developer.apple.com/documentation/visionos
- JSON endpoint: https://developer.apple.com/tutorials/data/documentation/visionos.json
- RealityKit: https://developer.apple.com/documentation/realitykit
- ARKit: https://developer.apple.com/documentation/arkit
- Reality Composer Pro / tools: https://developer.apple.com/augmented-reality/tools/

## 3. WWDC sessions
- Video index (filter "visionOS"): https://developer.apple.com/videos/
- Especially: "Principles of spatial design," "Design for spatial user
  interfaces," "Get started with building apps for spatial computing,"
  3D-content optimization sessions.
