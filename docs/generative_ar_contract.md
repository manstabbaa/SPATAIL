# SPATAIL Generative AR — cross-machine contract (v0.1)

The loop: **iPhone prompt → PC job server → agent drives live Blender (MCP) →
USDZ → iPhone downloads → AR playback.**

This file is the single source of truth both machines build against. The Mac
builds the iOS client to it; the PC builds the job server + generator to it.
Reachability is via **Tailscale** (PC has a stable tailnet address); no LAN/IP
assumptions, no firewall edits.

## Endpoints (PC job server)

Base URL (example): `http://<pc-tailnet-name>:8787`

### `POST /jobs`
Create a generation job.
```jsonc
// request
{ "prompt": "a bouncing red rubber ball", "client": "ios" }
// response 200
{ "id": "job_ab12cd", "status": "queued" }
```

### `GET /jobs/{id}`
Poll job state. Client polls every ~2s until `done` or `error`.
```jsonc
// response 200
{
  "id": "job_ab12cd",
  "status": "queued | running | done | error",
  "stage": "modeling | animating | exporting | ready",   // human-readable progress
  "message": "optional detail or error text",
  "usdz_url": "/artifacts/job_ab12cd.usdz",               // present iff status==done
  "metadata_url": "/artifacts/job_ab12cd_metadata.json"   // present iff status==done (optional)
}
```

### `GET /artifacts/{file}`
Serve generated bytes. `Content-Type: model/vnd.usdz+zip` for `.usdz`,
`application/json` for metadata. URLs returned above may be relative (resolve
against base URL) or absolute.

## Status lifecycle
```
queued ──► running ──► done
                  └──► error   (message explains why; client shows it, lets user retry)
```
- A job that never reaches `running` within ~120s, or `running` >5min, the client
  surfaces a timeout and offers retry. Server should still finish/clean up.

## Generated artifact requirements (so AR playback "just works")
- **Format:** binary USDZ, single file.
- **Up axis / units:** Y-up, meters (`metersPerUnit = 1`) — matches existing studio output.
- **Animation:** baked, looping. Object-transform tracks are fine (RealityKit
  surfaces them — verified). The app plays **every** clip in the hierarchy.
- **Scale/footprint:** object should fit a tabletop demo by default (roughly ≤ 1 m
  in its largest dimension) OR provide a bbox in metadata so the client can scale.
- **Metadata (optional but preferred):** reuse the existing shape from
  `studio/out/<exhibit>_metadata.json` (see `SpatailMetadata` in
  `ios/SpatailEducator/Sources/SpatailModel.swift`). Minimum useful fields:
  `title`, `bbox_yup_m` (min/max) or a footprint, and `animation` (fps/frames/loop).

## Generator (PC side, open-ended)
On a new job: an agent drives the **live first-party Blender MCP** (socket
`localhost:9876`, "MCP" extension, Auto Start) to model + animate from the prompt,
then exports USDZ into the artifacts dir and flips job status to `done` (or
`error` with a message). Keep Blender running ("live" end state). Headless bake
(`studio/ios_sync.py`) remains the reference for export settings.

## Versioning
Bump the `v0.x` in this header on any breaking change; both clients check it in
